class api_db_utils.APIDB¶
classmethod _APIDB__get(o, size=-1, paginated=False, page=0, count_only=False, **kwargs)¶

Implements the get of the query or of a list of objects.

It accepts server parameters to cover various scopes:

# get all members
cls.__get(o)
# get 10 trainers
cls.__get(o, 10)
# returns the number of members
cls.__get(o, count_only=True)
# get paginated results,for paginated
cls.__get(o, paginated=True)
# can also specify size (of the page) and page(starting point)
cls.__get(o, paginated=True,size=5,page=1)

The function can be used to retrieve:

  • Query,
  • list of Key (see _APIDB__get_multi_if_needed())
  • or relational data, belonging to a M2M relationship with a support table. (see _APIDB__get_relation())
Parameters:
  • o – the object
  • paginated – if the result has to be paginated
  • size – the size of the page or of the number of elements to retreive
  • page – the starting page number
  • count_only – if true, return the count
  • kwargs – remaining args that are generally used for the relationship, thus they can be ‘projection’ and ‘merge’
Returns:

  • a list of objects (if paginated = False)
  • a list of object, and the total number of elements (if paginated = True)
  • the total number of elements (if count_only = True)

classmethod _APIDB__get_relation_if_needed(result, projection=None, merge=None)¶

it gets the results from a list of relation that comes from the support table of an M2M rel.

this function automatically gets the relation using the projection parameter. If the merge value (string) is specified, it adds to the resulting object the support table in that field

CourseSubscription with projection = member will return a list of user``s. If the ``merge is set, and for example it’s set as merge="relation" then each user in the list will have relation property that is its CourseSubscription object. This property can be accessed via user.relation and it’s not stored into the db.

Parameters:
  • result – the result of the query
  • projection – the field to use for getting back the data
  • merge – the field where the relation is added (optional)
Returns:

the list of objects
